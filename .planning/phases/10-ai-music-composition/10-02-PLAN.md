    **Exported component interface:**
    ```ts
    interface MusicPickerDialogProps {
      open: boolean;
      onClose: () => void;

      onTrackAdded: (track: {
        id: string;
        name: string;
        type: "bgm";
        url: string;
        startTime: number;
        endTime: number | null;
        volume: number;
      }) => void;
    }

    export function MusicPickerDialog(props: MusicPickerDialogProps): JSX.Element
    ```

    **Internal types (not exported):**
    ```ts
    interface PixabayTrack {
      id: number;
      title: string;
      artist: string;
      url: string;
      previewUrl: string;
      duration: number;
    }
    ```

    **State:**
    - `activeTab: "search" | "upload"` — default `"search"`
    - `query: string` — search input text
    - `genre: string` — genre filter, default `""`
    - `tracks: PixabayTrack[]` — Pixabay search results
    - `searching: boolean` — loading state for search
    - `searchError: string | null`
    - `playingId: number | null` — currently previewing track id
    - `audioRef: React.MutableRefObject<HTMLAudioElement | null>`
    - `addingId: number | null` — track being added (loading state per-row)
    - `uploadFile: File | null`
    - `uploadName: string`
    - `uploading: boolean`
    - `uploadError: string | null`

    **Genre options constant (internal):**
    ```ts
    const GENRES = ["", "beats", "cinematic", "corporate", "electronic", "folk", "hip-hop", "jazz", "lofi", "pop", "rock"];
    ```

    **Search handler (`handleSearch`):**
    - Sets `searching = true`, clears `searchError`.
    - Calls `fetch(\`/api/music/search?q=\${encodeURIComponent(query)}&genre=\${encodeURIComponent(genre)}\`)`.
    - On non-ok response, sets `searchError = "검색 중 오류가 발생했습니다"` and `searching = false`.
    - On success, parses `{ tracks }` and sets `tracks` state.
    - Runs on Enter keypress in the query input and on a search button click.

    **Preview handler (`handlePreview`):**
    - If `playingId === track.id`, pause and set `playingId = null`.
    - Otherwise, pause any existing `audioRef.current`, create `new Audio(track.previewUrl)`, set `audio.onended = () => setPlayingId(null)`, call `audio.play()`, update `audioRef.current` and `playingId`.

    **Add Pixabay track handler (`handleAddPixabay`):**
    - Sets `addingId = track.id`.
    - POSTs to `/api/projects/${projectId}/audio-tracks` with JSON body:
      ```json
      { "type": "bgm", "name": "<track.title> — <track.artist>", "url": "<track.url>", "startTime": 0, "volume": 0.3 }
      ```
    - On success, calls `props.onTrackAdded(response.track)` then `props.onClose()`.
    - On error, shows a toast or `console.error` (toast preferred if `sonner` is available, otherwise just log).
    - Clears `addingId` in finally.

    **Upload handler (`handleUpload`):**
    - Guard: `uploadFile` must not be null. Validate MIME type is `audio/mpeg` or `audio/wav`. Max file size 50 MB (52_428_800 bytes). On violation, set `uploadError`.
    - Sets `uploading = true`, clears `uploadError`.
    - Builds `FormData` with fields: `file`, `type = "bgm"`, `name = uploadName || uploadFile.name.replace(/\.[^.]+$/, "")`, `startTime = "0"`, `volume = "0.3"`.
    - POSTs multipart to `/api/projects/${projectId}/audio-tracks`.
    - On success, calls `props.onTrackAdded(response.track)` then `props.onClose()`.
    - On error, sets `uploadError = "업로드 중 오류가 발생했습니다"`.
    - Clears `uploading` in finally.

    **Dialog close handler:** pause `audioRef.current`, reset `playingId`, call `props.onClose()`.
