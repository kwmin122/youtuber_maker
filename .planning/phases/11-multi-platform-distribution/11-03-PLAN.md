<objective>
This plan wires together the OAuth layer (Plan 01) and the upload workers (Plan 02) through an updated upload API route and an overhauled upload UI. The POST `/api/projects/[id]/upload` route is extended to accept `platforms: ("youtube" | "tiktok" | "reels")[]` and enqueue one independent BullMQ job per selected platform (D-11, D-12, D-14). The `PlatformSelector` component is replaced with a multi-checkbox variant. The `UploadPanel` is updated to track per-platform job status and show per-platform badges (D-13). The analytics page upload history table gains a platform column with color-coded badges for TikTok and Reels rows (D-13).

Requirements fulfilled: MULTI-01, MULTI-02
Phase goal contribution: Closes the loop so users can select YouTube + TikTok + Instagram simultaneously from a single upload dialog, see each platform's live status, and view cross-platform upload history in analytics.
Complexity: M
</objective>
