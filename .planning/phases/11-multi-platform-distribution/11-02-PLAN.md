Import: `spawn` from `"child_process"`, `writeFile`, `readFile`, `unlink`, `mkdtemp` from `"fs/promises"`, `join` from `"path"`, `tmpdir` from `"os"`.

Define and export platform limits:
```
export const PLATFORM_LIMITS = {
  youtube: { maxDurationSeconds: 60,  aspectRatio: "9:16" },
  tiktok:  { maxDurationSeconds: 180, aspectRatio: "9:16" },
  reels:   { maxDurationSeconds: 90,  aspectRatio: "9:16" },
} as const satisfies Record<import("@/lib/distribution/types").UploadPlatform, { maxDurationSeconds: number; aspectRatio: string }>
```

Export interface `VideoFormatResult { buffer: Buffer; durationSeconds: number; wasConverted: boolean }`.

Export async function `probeVideoFormat(buffer: Buffer): Promise<{ durationSeconds: number; width: number; height: number }>`:
- Write buffer to a temp file in `mkdtemp(join(tmpdir(), "vf-"))`.
- Spawn `ffprobe -v quiet -print_format json -show_streams -show_format <tempFile>`.
- Parse JSON output, extract `format.duration` (as float), and the first video stream's `width` and `height`.
- Clean up temp file. Return `{ durationSeconds, width, height }`.

Export async function `ensurePlatformFormat(buffer: Buffer, platform: import("@/lib/distribution/types").UploadPlatform): Promise<VideoFormatResult>`:
- Call `probeVideoFormat(buffer)` to get `{ durationSeconds, width, height }`.
- Check aspect ratio: if `width / height !== 9/16` (allow ±1% tolerance: `Math.abs(width/height - 9/16) > 0.01`), needs conversion.
- Check duration: if `durationSeconds > PLATFORM_LIMITS[platform].maxDurationSeconds`, needs trimming.
- If no conversion needed, return `{ buffer, durationSeconds, wasConverted: false }`.
- If conversion needed: build FFmpeg args. For 9:16 letterbox: `-vf "scale=1080:1920:force_original_aspect_ratio=decrease,pad=1080:1920:(ow-iw)/2:(oh-ih)/2:black"`. For duration trim: append `-t {maxDurationSeconds}` before output. Use temp input + output files, spawn FFmpeg, wait for exit code 0, read output buffer, clean up. Return `{ buffer: outputBuffer, durationSeconds: Math.min(durationSeconds, maxDuration), wasConverted: true }`.
- Throw descriptive error if FFmpeg exits non-zero.

Export async function:
```typescript
export async function uploadVideoToTikTok(params: {
  accessToken: string;
  videoBuffer: Buffer;
  title: string;
  description: string;
  privacyLevel: "PUBLIC_TO_EVERYONE" | "MUTUAL_FOLLOW_FRIENDS" | "SELF_ONLY";
  onProgress?: (percent: number) => void;
}): Promise<{ tiktokVideoId: string; videoUrl: string }>
```

Implementation uses TikTok Content Posting API v2 (direct post flow):

Step 1 — Initialize upload: POST `https://open.tiktokapis.com/v2/post/publish/video/init/` with headers `{ Authorization: "Bearer {accessToken}", "Content-Type": "application/json; charset=UTF-8" }` and body:
```json
{
  "post_info": {
    "title": "<title truncated to 150 chars>",
    "description": "<description truncated to 2200 chars>",
    "privacy_level": "<privacyLevel>",
    "disable_duet": false,
    "disable_stitch": false,
    "disable_comment": false,
    "video_cover_timestamp_ms": 1000
  },
  "source_info": {
    "source": "FILE_UPLOAD",
    "video_size": <videoBuffer.length>,
    "chunk_size": <videoBuffer.length>,
    "total_chunk_count": 1
  }
}
```
Parse response JSON: `data.publish_id` and `data.upload_url`.

Step 2 — Upload video bytes: PUT to `upload_url` with headers `{ "Content-Type": "video/mp4", "Content-Length": videoBuffer.length, "Content-Range": "bytes 0-{end}/{total}" }` where `end = videoBuffer.length - 1` and `total = videoBuffer.length`. Body is the raw Buffer. Call `onProgress(50)` after completion.

Step 3 — Poll publish status: GET `https://open.tiktokapis.com/v2/post/publish/status/fetch/?fields=status,fail_reason` with header `Authorization: "Bearer {accessToken}"` and body `{ "publish_id": publishId }` (note: TikTok status fetch uses POST with JSON body, not GET with query). Retry up to 12 times with 5-second delay between attempts. When `data.status === "PUBLISH_COMPLETE"`, extract `data.publicaly_available_post_id[0]` as `tiktokVideoId`. When `data.status === "FAILED"`, throw `Error("TikTok publish failed: " + data.fail_reason)`. Call `onProgress(100)` on completion.

Return `{ tiktokVideoId, videoUrl: "https://www.tiktok.com/@me/video/" + tiktokVideoId }`.

Wrap errors in descriptive `Error("TikTok upload failed: " + message)` messages.

Instagram Graph API Reels upload is a 2-step container create → publish flow:

Step 1 — Create container: POST `https://graph.instagram.com/v22.0/{igUserId}/media` with body (application/json): `{ "media_type": "REELS", "video_url": videoUrl, "caption": caption, "access_token": accessToken }`. Parse response for `id` as `containerId`. Call `onProgress(20)`.

Step 2 — Poll container status: GET `https://graph.instagram.com/v22.0/{containerId}?fields=status_code,status&access_token={accessToken}`. Poll every 5 seconds, up to 24 attempts (2 minutes max). Status `"FINISHED"` means ready to publish. Status `"ERROR"` throws. Call `onProgress(70)` when finished.

Step 3 — Publish: POST `https://graph.instagram.com/v22.0/{igUserId}/media_publish` with body `{ "creation_id": containerId, "access_token": accessToken }`. Parse response for `id` as `reelsVideoId`. Call `onProgress(100)`.

Return `{ reelsVideoId, videoUrl: "https://www.instagram.com/reels/" + reelsVideoId }`.

Wrap all errors in `Error("Instagram upload failed: " + message)`.
