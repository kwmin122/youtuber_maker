## Goal

Ship the full user-facing avatar flow inside an existing project's Media tab, using the APIs from 08-02 (library), 08-03 (reference upload), and the worker job from 08-04 (lipsync generation). After this plan a user can pick an avatar, set layout, generate lipsync for all scenes, and re-render the final export with the PIP overlay baked in.

## Scope

1. Restructure the existing Media tab (currently a flat set of scene cards) into 4 sub-tabs: `Image | Video | Audio | Avatar`.
2. Avatar library grid component with filter controls.
3. Reference photo upload flow that wires together: file picker → `POST /api/avatar/assets/upload-url` → PUT to signed URL → client-side SHA-256 → `POST /api/avatar/assets` (with consent modal from 08-03).
4. Per-scene avatar override dropdown + layout picker.
5. Project-wide default avatar selector.
6. "Generate for All Scenes" batch enqueue.
7. "Regenerate" per-scene button.
8. Cost-estimate banner (HeyGen ~$1.00/min, D-ID ~$0.10/min).
9. Longform-clip scene guard modal.
10. Polling progress per scene via the jobs API.
11. RTL tests for the grid, upload flow, layout picker, generate button.

### Task 4: Reference upload flow

**File**: `src/components/project/avatar-reference-upload.tsx` (NEW)

**Action**: the full upload flow:

1. File input accepts `image/jpeg,image/png,image/webp` (MIME whitelist matching the bucket SQL).
2. Client-side: reject files > 20 MB.
3. Client-side: compute SHA-256 via `crypto.subtle.digest` → hex.
4. Show consent modal (`AvatarConsentModal` from 08-03).
5. On confirm:
   a. `POST /api/avatar/assets/upload-url` with `{ ext }` → get `{ signedUrl, storagePath }`.
   b. `PUT signedUrl` with the file bytes as body, `content-type: <mime>`.
   c. `POST /api/avatar/assets` with `{ storagePath, imageHash, consent: true }` → row id.
6. Emit `onUploadComplete(rowId)`.

```tsx
"use client";

import { useState, useRef } from "react";
import { Button } from "@/components/ui/button";
import { AvatarConsentModal } from "./avatar-consent-modal";

const MAX_BYTES = 20 * 1024 * 1024;
const ALLOWED = ["image/jpeg", "image/png", "image/webp"];

async function sha256Hex(buf: ArrayBuffer): Promise<string> {
  const digest = await crypto.subtle.digest("SHA-256", buf);
  return Array.from(new Uint8Array(digest))
    .map((b) => b.toString(16).padStart(2, "0"))
    .join("");
}

interface Props {
  onUploadComplete?: (assetId: string) => void;
}

export function AvatarReferenceUpload({ onUploadComplete }: Props) {
  const [file, setFile] = useState<File | null>(null);
  const [error, setError] = useState<string | null>(null);
  const [uploading, setUploading] = useState(false);
  const [consentOpen, setConsentOpen] = useState(false);
  const fileInputRef = useRef<HTMLInputElement>(null);

  function onPick(e: React.ChangeEvent<HTMLInputElement>) {
    const f = e.target.files?.[0] ?? null;
    setError(null);
    if (!f) return;
    if (!ALLOWED.includes(f.type)) {
      setError("지원하지 않는 파일 형식입니다. (JPG/PNG/WebP만 가능)");
      return;
    }
    if (f.size > MAX_BYTES) {
      setError("파일 크기가 20 MB를 초과합니다.");
      return;
    }
    setFile(f);
    setConsentOpen(true);
  }

  async function performUpload() {
    if (!file) return;
    setUploading(true);
    try {
      const ext = file.type === "image/jpeg" ? "jpg" : file.type === "image/png" ? "png" : "webp";

      // 1. Signed upload URL
      const urlRes = await fetch("/api/avatar/assets/upload-url", {
        method: "POST",
        headers: { "content-type": "application/json" },
        body: JSON.stringify({ ext }),
      });
      if (!urlRes.ok) throw new Error(`upload-url failed: ${urlRes.status}`);
      const { signedUrl, storagePath } = (await urlRes.json()) as {
        signedUrl: string;
        storagePath: string;
      };

      // 2. Compute hash
      const bytes = await file.arrayBuffer();
      const imageHash = await sha256Hex(bytes);

      // 3. PUT to signed URL
      const putRes = await fetch(signedUrl, {
        method: "PUT",
        headers: { "content-type": file.type, "x-upsert": "true" },
        body: bytes,
      });
      if (!putRes.ok) throw new Error(`PUT failed: ${putRes.status}`);

      // 4. Register in DB
      const createRes = await fetch("/api/avatar/assets", {
        method: "POST",
        headers: { "content-type": "application/json" },
        body: JSON.stringify({ storagePath, imageHash, consent: true }),
      });
      if (!createRes.ok) {
        const errBody = await createRes.json().catch(() => ({}));
        throw new Error(`asset create failed: ${errBody.error ?? createRes.status}`);
      }
      const { id } = (await createRes.json()) as { id: string };
      onUploadComplete?.(id);
      setFile(null);
      if (fileInputRef.current) fileInputRef.current.value = "";
    } catch (e) {
      setError((e as Error).message);
    } finally {
      setUploading(false);
    }
  }

  return (
    <div className="space-y-3">
      <input
        ref={fileInputRef}
        type="file"
        accept={ALLOWED.join(",")}
        onChange={onPick}
        data-testid="avatar-file-input"
        className="block text-sm"
      />
      {file && <p className="text-sm text-muted-foreground">선택됨: {file.name} ({(file.size / 1024 / 1024).toFixed(1)} MB)</p>}
      {error && <p className="text-sm text-destructive">{error}</p>}
      {uploading && <p className="text-sm text-muted-foreground">업로드 중...</p>}
      <AvatarConsentModal
        open={consentOpen}
        onOpenChange={setConsentOpen}
        onConfirm={performUpload}
      />
    </div>
  );
}
```

### Task 6: API for persisting scene avatar settings

**File**: `src/app/api/scenes/[id]/avatar/route.ts` (NEW)

**Action**: `PATCH` endpoint that updates `scenes.avatarPresetId` and `scenes.avatarLayout` for a given sceneId. Ownership check: join scenes→scripts→projects→userId BEFORE writing. Mirror the IDOR-guard pattern in `src/app/api/jobs/route.ts` lines 57-88.

```ts
import { NextResponse } from "next/server";
import { z } from "zod";
import { eq } from "drizzle-orm";
import { db } from "@/lib/db";
import { scenes, scripts, projects } from "@/lib/db/schema";
import { getServerSession } from "@/lib/auth/get-session";

const bodySchema = z.object({
  avatarPresetId: z.string().uuid().nullable().optional(),
  avatarLayout: z
    .object({
      enabled: z.boolean(),
      position: z.enum(["bottom-right", "bottom-left", "center", "top-right", "fullscreen"]),
      scale: z.number().min(0.1).max(1),
      paddingPx: z.number().int().min(0).max(128),
    })
    .nullable()
    .optional(),
});

type Params = { params: Promise<{ id: string }> };

export async function PATCH(request: Request, { params }: Params) {
  const session = await getServerSession();
  if (!session?.user) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }
  const { id } = await params;

  const body = await request.json().catch(() => null);
  const parsed = bodySchema.safeParse(body);
  if (!parsed.success) {
    return NextResponse.json({ error: "Invalid request", details: parsed.error.flatten() }, { status: 400 });
  }

  // Ownership join
  const [row] = await db
    .select({
      sceneId: scenes.id,
      userId: projects.userId,
    })
    .from(scenes)
    .leftJoin(scripts, eq(scripts.id, scenes.scriptId))
    .leftJoin(projects, eq(projects.id, scripts.projectId))
    .where(eq(scenes.id, id))
    .limit(1);

  if (!row) {
    return NextResponse.json({ error: "not found" }, { status: 404 });
  }
  if (row.userId !== session.user.id) {
    return NextResponse.json({ error: "forbidden" }, { status: 403 });
  }

  await db
    .update(scenes)
    .set({
      avatarPresetId: parsed.data.avatarPresetId ?? null,
      avatarLayout: parsed.data.avatarLayout ?? null,
      updatedAt: new Date(),
    })
    .where(eq(scenes.id, id));

  return NextResponse.json({ updated: true });
}
```

### Task 7: Polling scene avatar status

**Action**: small client hook that accepts a list of scene ids, polls `GET /api/jobs?projectId=<projectId>` every 3 seconds, and returns a map `{ sceneId: { status, progress } }` based on the job rows whose `type === 'generate-avatar-lipsync'` and whose payload references the scene. The simplest path: include the sceneId in the job row's `payload` and filter client-side.

```ts
"use client";
import { useEffect, useState } from "react";

type JobRow = {
  id: string;
  type: string;
  status: string;
  progress: number;
  // Note: GET /api/jobs does not currently return payload. Either
  // extend the API to include `payload.sceneId`, or add a new
  // endpoint GET /api/avatar/scene-progress?projectId=X that returns
  // a map. Prefer the latter to avoid leaking payload shape.
};

export function useSceneAvatarJobs(projectId: string, pollMs = 3000) {
  const [progress, setProgress] = useState<Record<string, { status: string; progress: number }>>({});

  useEffect(() => {
    let cancel = false;
    async function tick() {
      const res = await fetch(`/api/avatar/scene-progress?projectId=${projectId}`).catch(() => null);
      if (!res?.ok || cancel) return;
      const body = (await res.json()) as Record<string, { status: string; progress: number }>;
      setProgress(body);
    }
    tick();
    const t = setInterval(tick, pollMs);
    return () => {
      cancel = true;
      clearInterval(t);
    };
  }, [projectId, pollMs]);

  return progress;
}
```

**File**: `src/app/api/avatar/scene-progress/route.ts` (NEW)

**Action**: `GET` endpoint that returns a map `{ sceneId: { status, progress } }` for the latest `generate-avatar-lipsync` job per scene in the project. Ownership-gated on `projects.userId`. Join `jobs` to the scenes (via jobs.projectId or via payload.sceneId — payload.sceneId is in a JSONB column, so use `payload->>'sceneId'`).

```ts
import { NextRequest, NextResponse } from "next/server";
import { sql, eq, and, desc } from "drizzle-orm";
import { db } from "@/lib/db";
import { jobs, projects } from "@/lib/db/schema";
import { getServerSession } from "@/lib/auth/get-session";

export async function GET(request: NextRequest) {
  const session = await getServerSession();
  if (!session?.user) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  const projectId = new URL(request.url).searchParams.get("projectId");
  if (!projectId) return NextResponse.json({ error: "projectId required" }, { status: 400 });

  const [proj] = await db.select().from(projects).where(eq(projects.id, projectId)).limit(1);
  if (!proj) return NextResponse.json({ error: "not found" }, { status: 404 });
  if (proj.userId !== session.user.id) {
    return NextResponse.json({ error: "forbidden" }, { status: 403 });
  }

  const rows = await db
    .select({
      sceneId: sql<string>`${jobs.payload}->>'sceneId'`,
      status: jobs.status,
      progress: jobs.progress,
      createdAt: jobs.createdAt,
    })
    .from(jobs)
    .where(
      and(
        eq(jobs.type, "generate-avatar-lipsync"),
        eq(jobs.projectId, projectId),
        eq(jobs.userId, session.user.id)
      )
    )
    .orderBy(desc(jobs.createdAt))
    .limit(200);

  const map: Record<string, { status: string; progress: number }> = {};
  for (const r of rows) {
    if (!r.sceneId) continue;
    if (!map[r.sceneId]) {
      map[r.sceneId] = { status: r.status, progress: r.progress };
    }
  }
  return NextResponse.json(map);
}
```

## Risks & Mitigations

- **Risk**: The scene-tab refactor accidentally removes existing image/video/audio functionality. **Mitigation**: make the sub-tab wrapper additive — move existing panels verbatim into the `Image`/`Video`/`Audio` sub-tab contents without modifying their internals. Run the full v1 test suite before merging.
- **Risk**: Client-side SHA-256 via `crypto.subtle` is slow for 20 MB files on low-end devices. **Mitigation**: worst case ~500ms, acceptable for a one-off consent flow. Show an "해싱 중..." indicator if needed.
- **Risk**: The scene-progress endpoint queries `jobs.payload->>'sceneId'` on every poll. **Mitigation**: the query is bounded by `projectId` + `userId`, so the worst case is O(N_scenes × poll_interval). For projects with < 20 scenes and a 3s poll, this is trivial load. Add an index on `jobs.projectId` if load becomes a concern (Phase 9+).
- **Risk**: Layout picker writes to `scenes.avatarLayout` directly, but the user may click "Generate" before the PATCH resolves. **Mitigation**: local state is the source of truth for the generate button; PATCH calls are optimistic fire-and-forget.
- **Risk**: Cost estimate uses `scenes.duration`, which Plan 08-04 overwrites after avatar generation — the pre-generate estimate may be wrong. **Mitigation**: the estimate is explicitly an "estimate"; post-generate the banner shows the actual cost. Document this in the banner copy ("예상").
